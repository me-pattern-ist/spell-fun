# Product Context

Before suggesting major new features or architectural changes, refer to the Product Requirements Document.

## Read the PRD
The core features, target audience, and current scope of the application are maintained in:
`doc/PRD.md`

Always ensure new additions align with the goals outlined in that document (e.g., keeping interactions simple for kids, relying on local AI generation over cloud APIs).
